# NLPrun Commands for Small County Experiments

This document contains the nlprun commands to run the small county experiments.

## Output Files

All output files (`.out`) from nlprun are saved to `/sailhome/salilg/tabpfn_data_scarcity/experiments/outfiles/`.

The Python logging goes to stderr, which nlprun captures in the `.out` files. There are no separate `.log` files created.

## Prerequisites

1. Create the output directory:
```bash
mkdir -p /sailhome/salilg/tabpfn_data_scarcity/experiments/outfiles
```

2. Create the county registry:
```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/scripts
python 00_create_county_registry.py
```

Or run via nlprun:
```bash
nlprun -q john -p standard -a pfn_env -n create_county_registry \
  -o /sailhome/salilg/tabpfn_data_scarcity/experiments/outfiles/create_county_registry.out \
  'cd /sailhome/salilg/tabpfn_data_scarcity/experiments/scripts && python 00_create_county_registry.py'
```

This creates `small_county_metadata.csv` with all counties in the 10-100 size bin.

## Experiment 1: Within-County Nested CV

For the within-county experiment, you need to run one job per county. The registry script will tell you how many counties there are.

### Single County Example

To run a single county (e.g., FIPS 1007):

```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/runners

nlprun -q jag -p standard -a pfn_env -g 1 -c 8 -r 32G \
  -n within-county-1007 \
  -o ../outfiles/within_county_1007.out \
  'python within_county_runner.py --fips 1007 --bin_name small --k_folds 5'
```

### Running Multiple Counties in Parallel

To run multiple counties in parallel (adjust the number based on available resources):

**Option 1: Launch 10 counties in parallel**
```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/runners

# Get list of counties from metadata (first 10 as example)
# FIPS codes will vary - replace with actual FIPS from small_county_metadata.csv

nlprun -q jag -p standard -a pfn_env -g 1 -c 8 -r 32G -n within-1007 -o ../outfiles/within_1007.out 'python within_county_runner.py --fips 1007 --bin_name small --k_folds 5' &
nlprun -q jag -p standard -a pfn_env -g 1 -c 8 -r 32G -n within-1011 -o ../outfiles/within_1011.out 'python within_county_runner.py --fips 1011 --bin_name small --k_folds 5' &
nlprun -q jag -p standard -a pfn_env -g 1 -c 8 -r 32G -n within-1013 -o ../outfiles/within_1013.out 'python within_county_runner.py --fips 1013 --bin_name small --k_folds 5' &
# ... continue for other counties
wait
```

**Option 2: Use a helper script (recommended)**

See `02_launch_within_county_nlprun.sh` which automatically launches jobs for all counties.

```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/scripts
bash 02_launch_within_county_nlprun.sh --max-parallel 10
```

This will launch up to 10 jobs in parallel, and as they complete, launch more until all counties are processed.

## Experiment 2: Cross-County Pooled Training

For the cross-county experiment, you need to run multiple iterations per county.

### Single County-Iteration Example

To run one iteration for one county (e.g., FIPS 1007, iteration 0):

```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/runners

# First, get the list of all small county FIPS codes (comma-separated)
# This will be generated by the registry script or helper script
FIPS_LIST="1007,1011,1013,..."  # Replace with actual list

nlprun -q jag -p standard -a pfn_env -g 1 -c 8 -r 64G \
  -n cross-county-1007-iter0 \
  -o ../outfiles/cross_county_1007_iter0.out \
  "python cross_county_runner.py --target_fips 1007 --fips_list $FIPS_LIST --iteration 0 --bin_name small"
```

### Running Multiple Iterations

**Recommended: Use helper script**

```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/scripts
bash 03_launch_cross_county_nlprun.sh --max-parallel 10
```

This script will:
1. Read all small counties from metadata
2. For each county, launch 10 iterations (configurable)
3. Manage parallelism to stay within your limit

## Resource Requirements

### Within-County Experiment
- **GPU**: 1 GPU per job
- **CPU**: 8 cores recommended
- **Memory**: 32GB
- **Time**: ~2-4 hours per county (depends on size and number of folds)
- **Total jobs**: ~number of small counties (likely 50-200)

### Cross-County Experiment
- **GPU**: 1 GPU per job
- **CPU**: 8 cores recommended
- **Memory**: 64GB (larger because of pooled data)
- **Time**: ~1-3 hours per (county, iteration) pair
- **Total jobs**: ~number of counties × 10 iterations (likely 500-2000)

## Monitoring Jobs

Check running jobs:
```bash
nlpjobs
```

Check output of a specific job:
```bash
tail -f /sailhome/salilg/tabpfn_data_scarcity/experiments/outfiles/within_county_1007.out
```

## Aggregating Results

After all jobs complete, aggregate the results:

```bash
cd /sailhome/salilg/tabpfn_data_scarcity/experiments/scripts
python 04_aggregate_results.py --experiment within_county
python 04_aggregate_results.py --experiment cross_county
```

## Notes

- Adjust `-q jag` and `-p standard` based on your cluster's queue/partition names
- The `-a pfn_env` assumes you have a conda environment named `pfn_env` with all dependencies
- You can adjust parallelism by changing the `--max-parallel` argument
- Output files (`.out`) are saved to `/sailhome/salilg/tabpfn_data_scarcity/experiments/outfiles/`
- Results (CSV files) are saved to `/sailhome/salilg/tabpfn_data_scarcity/results/{within_county,cross_county}/`
